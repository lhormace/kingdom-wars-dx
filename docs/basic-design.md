# Kingdom Wars DX 基本設計書

## 1. 文書の目的

本書は `Kingdom Wars DX` の現行仕様を、利用者視点とシステム全体視点で整理するための基本設計書である。  
対象は 2026-04-04 時点の実装であり、ブラウザ上で動作するシングルプレイのタクティカルアクションゲームを前提とする。

## 2. システム概要

- 名称: `Kingdom Wars DX`
- 形態: ブラウザ動作のシングルページゲーム
- 主エンジン: `Phaser`
- 補助ライブラリ: `Babylon.js`, `PixiJS`, `phina.js`, `enchant.js`, `Kiwi.js`
- 配信方式: GitHub Pages
- ライブラリ管理: `vendor/` 配下のローカル同梱

本ゲームは、王を中心とした王国軍を操作して敵城へ進軍し、捕虜の救出、聖剣の回収、ボス撃破を経てステージクリアを目指す。

## 3. ゲームコンセプト

- ケルト神話風の意匠をもつ深緑と金の戦場表現
- ライン戦術と隊列維持を重視した前進型ゲームプレイ
- 王、兵士、騎士、大魔法使い、僧侶の役割分担
- 進軍、戦闘、救出、回復、魔法、イベントの複合的な判断

## 4. プレイヤー体験

- プレイヤーは王を直接操作する
- 味方隊列は王の背後に追従する
- 敵と接敵すると戦闘が発生する
- 捕虜を救出すると味方兵力が増加する
- 聖剣を取得すると強敵への優位性が生まれる
- ステージ進行に伴い敵量と強敵の圧力が増す

## 5. 画面構成

### 5.1 メイン画面

- ヒーロー領域
  - タイトル
  - バージョン表示
  - 開始ボタン
  - やり直しボタン
- ゲーム画面
  - Phaser による本編描画
- サイドパネル
  - ルール説明
  - 操作説明
  - 凡例
  - 六王国連携祭壇

### 5.2 六王国連携祭壇

主ゲームとは別に、戦況を補助表示するための可視化群を持つ。

- `Babylon.js`: 儀式的な 3D 演出
- `PixiJS`: 敵圧と士気の可視化
- `phina.js`: 王国コア表現
- `enchant.js`: 進軍リレー演出
- `Kiwi.js`: 戦況メッセージ演出

## 6. ゲーム進行

### 6.1 状態遷移

- `title`
  - タイトル表示
  - 開始待ち
- `playing`
  - 通常進軍中
- `paused`
  - 一時停止
- `clear`
  - ステージクリア
- `gameover`
  - 敗北

### 6.2 ステージの流れ

1. ステージ初期化
2. マップ生成
3. 敵、捕虜、聖剣配置
4. 王国軍出撃
5. 進軍、戦闘、救出、イベント発生
6. ボス撃破後に城へ侵入するとステージクリア
7. 次ステージへ進行

## 7. プレイヤー操作

- 移動: `W`, `A`, `S`, `D` / 矢印キー
- 魔法: `F`
- ステージ再挑戦: `R`
- 開始 / 再開 / 一時停止 / 次へ: `Enter`, `Space`, ボタン

## 8. 主要ゲーム要素

### 8.1 味方

- 王

## 10. システム構成

- `index.html`
  - エントリポイント
  - バージョン読込
  - スクリプト起動
- `game.js`
  - 本編ゲーム実装
- `src/gameEnhancements.js`
  - 六王国連携祭壇
- `src/libraryAdapters.js`
  - ライブラリデモの個別初期化
- `vendor/`
  - ローカル同梱ライブラリ
- `version.txt`
  - 表示用バージョン情報

## 11. 基本方針

- 主ゲームロジックは `Phaser` 上に集約する
- 補助ライブラリは本編支援表示や演出へ利用する
- CDN 依存は持たず、ライブラリはローカルに固定する
- GitHub Pages のキャッシュ回避を考慮する

## 12. 想定利用環境

- PC ブラウザ
- キーボード操作前提
- ネット接続は初回配信取得時のみ前提

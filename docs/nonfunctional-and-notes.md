# Kingdom Wars DX 非機能要件・注意事項設計書

## 1. 目的

本書は、機能仕様以外で考慮すべき非機能要件、運用上の注意、保守上の注意を整理する。

## 2. 非機能要件

### 2.1 可用性

- GitHub Pages 上で安定して表示できること
- 初回読込後はライブラリ取得のために外部 CDN を必要としないこと

### 2.2 性能

- 一般的な PC ブラウザ上でプレイ可能な描画速度を維持すること
- 入力から移動までの体感遅延を抑えること
- 同一フレーム内での不要な多重処理を避けること

### 2.3 保守性

- ライブラリを `vendor/` 配下に固定し、差分管理可能にすること
- バージョン表記を `version.txt` に集約すること
- version 更新を commit 前フックで自動化すること

### 2.4 可搬性

- 主要モダンブラウザで動作すること
- ローカルファイル依存ではなく、相対パス配信で成立すること

### 2.5 観測性

- 重大な不具合時に原因を追えるよう、必要に応じて一時的な診断表示やログ追加を許容すること
- ただし恒久的にはデバッグ表示を画面へ残さないこと

## 3. セキュリティ / 配信上の注意

- 外部 CDN からの実行時スクリプト読込を避ける
- 配信ファイルは静的ホスティングで完結させる
- バージョン付き URL によりキャッシュ不整合を抑止する

## 4. UX 上の注意

- キーボード操作主体であるためフォーカス管理を重視する
- 開始ボタン押下後にゲーム画面へ操作主体が移ること
- 一時停止、再開、ゲームオーバー、クリア時の状態遷移が分かりやすいこと
- 戦況表示は華美すぎず、本編視認性を阻害しないこと

## 5. ライブラリ運用方針

- `Phaser` を主エンジンとして扱う
- 他ライブラリは補助演出、可視化、検証用途に回す
- 本編ロジックを複数ライブラリへ分散しすぎない
- 同梱版ライブラリ更新時は互換性確認を行う

## 6. 既知の設計上の注意

- `game.js` に責務集中がある
- 入力処理は複数回の修正を経ているため、今後も変更時は回帰確認が必要
- 捕虜や聖剣の回収判定は厳密タイル一致を前提に維持する
- 表示補間とロジック座標の混同を避ける

## 7. 運用手順上の注意

### 7.1 バージョン更新

- `version.txt` は手動編集ではなく commit 前フックで更新される
- push 前に `version.txt` が変わる前提でレビューする

### 7.2 デバッグ

- 不具合の切り分け時は
  - 入力
  - 移動判定
  - 描画反映
  - 回収判定
  を分けて確認する

### 7.3 デプロイ

- `main` 反映後に GitHub Pages の配信反映まで多少のタイムラグを見込む
- キャッシュ差異がある場合は `version.txt` と URL の `v` を確認する

## 8. 今後の非機能改善候補

- 自動テスト導入
- ESLint / Formatter 導入
- 仕様変更履歴の記録
- ランタイム診断モードの明示的切替
- 操作レスポンス計測
